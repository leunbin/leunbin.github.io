BACKEND
앨리스 부트캠프 2차 팀 프로젝트로 제작한 웹 애플리케이션입니다.
사용자가 가진 재료를 이용해 레시피를 검색하고 레시피 작성 및 이미지 업로드, 좋아요와 북마크 등 사용자 상호작용 기능을 구현했습니다.
🧩 My Role & Key Work

회원 기능 : 로그인, 회원가입, 탈퇴, 마이페이지, 북마크 등 핵심 인증/인가 기능 구현

데이터 설계 : Mongoose Schema 기반 데이터베이스 구조 설계 및 관계 설정

문서화 : Notion, Google Sheets, Swagger를 이용한 API 명세 작성 및 협업 효율성 강화

인증 시스템 : JWT 기반 로그인 및 세션 관리, 이메일 인증 프로세스 설계

💭 Retrospective

두 번째 엘리스 프로젝트로 4명의 프론트엔드와 2명의 백엔드가 함께 협업하는 환경에서 요구사항 조율과 일정 관리의 중요성을 크게 느꼈습니다. 프론트 쪽에서 추가 기능 요청이 많았지만 한정된 기간 내에서 우선순위를 정해 개발 범위를 명확히 조율하여 실무에 가까운 협업을 경험했습니다.

JWT 토큰 기반의 회원 인증과 메일 인증(NodeMailer)을 적용해 사용자 인증 절차를 강화하여 서버와 클라이언트 간의 데이터 흐름과 인증 구조를 직접 이해할 수 있었습니다. 또한 사용자 상호작용 기능(좋아요·북마크)을 통해 상태 관리와 비동기 통신 구조를 경험했습니다.

단순히 기능 구현을 넘어 주어진 상황에서 팀원 간의 의견 공유를 통해 효율적으로 프로젝트를 완성하였습니다. 이후 다른 프로젝트에서도 API 명세와 필수 기능 중심의 협업을 실천하게 되었습니다.

💭 Retrospective

첫 프론트엔드 경험으로 이전에 백엔드만 담당했을 때 API 명세서나 스키마를 제때 전달하지 못해 프론트 팀이 얼마나 불편했을지 미처 실감하지 못했었습니다. 직접 프론트에서 API 연동을 경험하며 개발 단계마다 명확한 커뮤니케이션과 문서화의 중요성을 체감했습니다.

또한 팀 내에서 자신의 업무를 도와달라는 요청이 있어 처음에는 수용하여 완성도에 차질이 없도록 협력했지만 추가 요청에 대해서는 제 일정에 영향을 줄 수 있다고 판단해 다른 팀원들과 조율하여 우선순위를 명확히 조정했습니다. 이 과정을 통해 협업은 단순한 소통이 아닌 서로의 책임과 일정을 존중하며 조율하는 과정임을 배웠습니다.
